生命周期

钩子函数的变化

  1. beforeCreate –> 组件实例刚被创建，没有任何东西
  2. created –> 实例完成了属性、方法等的绑定，是实实在在的实例了，但是 DOM 还没有
  3. beforeMount –> 模板编译之前
  4. mounted –> 模板编译完成 也就是之前的 ready
  5. beforeUpdate –> 组件更新之前
  6. update –> 组件更新完毕
  7. beforeDestroy –> 组件销毁前
  8. destoryed –> 组件销毁后

循环

  1. 2.0里面默认的可以添加重复的数据
  2. v-for="( val, index/key )" in data
  3. 提升循环性能 :key="index/key" 相当于 1.0 里面的 track-by="index/key"

自定义键盘指令

2.0 变成了 Vue.config.keyCodes.ctrl = 17

过滤器

2.0 内置过滤器全部删除了,自己通过 JS 实现
json 不需要再写 | json 了，会直接自己转换了
自定义过滤器的传参发生了变化

组件通信

父组件和子组件

1.0 子组件可以更改父组件的信息，而且可以是同步的（ sync ）
2.0 不允许了，并且在子组件不允许直接更改子组件的数据。

如果需要 1.0 同步的效果，就需要每次传一个对象给子组件 （推荐方式）

如果只是为了不报错，可以使用 mounted 中转一下。

动态路由

当不同 id 的用户访问 user 页面的时候，我们需要渲染出相同的页面，那么就需要配置动态路由

注意：这里渲染的组件只是对组件的复用，而不是又创建了一个,所以组件的生命周期钩子函数不会被再次调用，如果想监视路径的变化，可以使用 watch $route 对象

router-link

属性

to 表示目标路由的连接，当被点击的时候，内部会立刻将 to 的值传递给 router.push(),to 的值可以是一个字符串，也可以是描述目标位置的对象

replace 设置了该属性，被点击时，会调用 router.replace（）不会调用 router.push()方法，所有不会被添加到浏览器的实例记录里

append 设置了该属性，相当于将该链接的路由添加到上一次路由的后面

tag 因为 router-lick 默认将链接渲染为 a 标签,设置该属性可以定义其渲染为哪种标签,点击的效果都还在

<router-lick :to="home" tag="li"></router-link>

active-class 设置链接激活时候使用的类名 默认值是 router-link-active

激活时的CSS类名应用到外层元素
