js判断是在APP内还是H5

很多移动端的项目是用H5打包的APP，H5再用，APP也在用。但是在APP中的一些部分要显示的和H5不一样，于是就要用到了来判断是APP还是H5.

  • 判断是否是在APP内

  • 关于判断的问题
    IOS的判断是通过 navigator.userAgent 来判断的，没有什么问题. 但是安卓的要说一下了。这里就牵扯到了Javascript与Android Native的交互
    在打包APP的时候，安卓那边可以通过一个addJavascriptInterface方法来让JS这边可以调用安卓的那边定义的方法(详细请点击上面连接)

    // 示例使用 addJavascriptInterface 方法，webView 和 new WebAppInterface 是安卓那边定义的，详情查看上述连接
// 这里重点关注后面的 "android"  当然名字可以随便起，只要别重复就行
webView.addJavascriptInterface(new WebAppInterface(this), "android"); //"android"实在JavaScript中调用的名字

    写了上述的一句后，相当于把android挂载了window对象下面，所以，JS这边可以通过检测有没有这个方法来判断是否是在AndroidAPP中。即上面所写的 window.android
